# Nat 5 Computing v8.2 setup

This version keeps the existing school/class pilot working and adds:

- individual pupil sign-in by real email address;
- permanent complimentary teacher and pupil entitlements;
- Full Course and Exam Accelerator route switching;
- class-free progress, revision, games and spaced practice for individual pupils;
- password-reset email support for individual accounts;
- Firestore rules that permit individual learning only when an active entitlement exists.

No Blaze upgrade or Cloud Functions are required for these four complimentary accounts.

## Create each founding teacher

The teacher continues to sign in with Google.

1. In Firestore, make sure this document exists:

   `teacherInvites/{teacher-email-in-lowercase}`

   Suggested fields:

   ```text
   active: true
   accessType: "founding-complimentary"
   permanent: true
   ```

2. Ask the teacher to sign in once through the Teacher card. This creates the normal `users/{uid}` profile if it does not already exist.
3. Copy the teacher UID from Firebase Authentication.
4. Create:

   `users/{uid}/entitlements/nat5-computing`

   with:

   ```text
   courseId: "nat5-computing"
   licenceType: "teacher"
   source: "founding-complimentary"
   status: "active"
   permanent: true
   includesClassAccess: true
   includesFullCourse: true
   includesExamAccelerator: true
   grantedAt: [Firestore timestamp]
   ```

The teacher dashboard will display **Founding complimentary access — Permanent**.

## Create each founding individual pupil

Individual account creation is deliberately administrator-controlled while the project remains on Spark.

1. Open Firebase Authentication.
2. Choose **Add user**.
3. Enter the pupil's real email address and a temporary password.
4. Copy the generated UID.
5. Create `users/{uid}` with:

   ```text
   role: "student"
   accountType: "individual"
   displayName: "Pupil name"
   email: "pupil@example.com"
   createdAt: [Firestore timestamp]
   ```

   Do not add `classId`, `className` or `joinCode`.

6. Create:

   `users/{uid}/entitlements/nat5-computing`

   with:

   ```text
   courseId: "nat5-computing"
   licenceType: "individual-student"
   source: "founding-complimentary"
   status: "active"
   permanent: true
   includesFullCourse: true
   includesExamAccelerator: true
   grantedAt: [Firestore timestamp]
   ```

7. Give the pupil the website address, email and temporary password.
8. They sign in through **Individual learner**. They can use the password-reset button if necessary.

## How individual progress is stored

School pupils continue using their existing class scope.

Individual pupils use a private scope:

```text
individual_{firebase-uid}
```

Their progress documents still use the existing collections, but the stored `classId` is this private scope. The rules only allow the matching authenticated pupil with an active entitlement to use it.

Full Course and Exam Accelerator use the same activity IDs and mastery records. Switching route does not reset or duplicate progress.

## Exam Accelerator in v8.2

The initial accelerator is generated from the existing course bank. Every unit is represented by up to four high-value activities:

1. rapid lesson or explanation;
2. practical/prediction/visual activity;
3. retrieval quiz or fluency practice;
4. exam-style or official-paper practice.

All accelerator topics are open, so pupils can move directly to urgent exam gaps. The Full Course remains sequential.

## Blaze later

When paid access launches, keep the same entitlement document structure. A trusted Cloud Function/Stripe webhook should create and update entitlement documents. Browser clients remain unable to grant, extend or alter their own licences.
