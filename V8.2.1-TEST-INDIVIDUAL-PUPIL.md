# Create and test an individual pupil account

This version keeps individual account creation administrator-controlled while the site is on Firebase Spark.

## 1. Create the Authentication user

In Firebase Console:

1. Open **Authentication**.
2. Open **Users**.
3. Choose **Add user**.
4. Enter a test email, for example `testpupil@example.com`.
5. Enter a temporary password of at least 8 characters.
6. Copy the new user's **UID**.

## 2. Create the user profile

In Firestore, create this document:

`users/{UID}`

Use these fields:

| Field | Type | Example value |
|---|---|---|
| `role` | string | `student` |
| `accountType` | string | `individual` |
| `displayName` | string | `Test Pupil` |
| `email` | string | `testpupil@example.com` |
| `createdAt` | timestamp | current date/time |

Do not add `classId`, `className` or `joinCode`.

## 3. Grant the permanent test licence

Under the same user document, create the subcollection and document:

`users/{UID}/entitlements/nat5-computing`

Use these fields:

| Field | Type | Value |
|---|---|---|
| `courseId` | string | `nat5-computing` |
| `licenceType` | string | `individual-student` |
| `source` | string | `founding-complimentary` |
| `status` | string | `active` |
| `permanent` | boolean | `true` |
| `includesFullCourse` | boolean | `true` |
| `includesExamAccelerator` | boolean | `true` |
| `grantedAt` | timestamp | current date/time |

## 4. Sign in

Open the website and use the **Individual learner** card—not the class pupil card.

Enter the email and temporary password created in Authentication.

## Common errors

- **Profile could not be found:** the `users/{UID}` document is missing or the UID is not exact.
- **No active Nat 5 Computing licence:** the entitlement document is missing, is not named `nat5-computing`, or `status` is not `active`.
- **Email or password not recognised:** the Authentication user was not created or the password is wrong.
- **Permission denied after sign-in:** deploy the included `firestore.rules` before testing.
